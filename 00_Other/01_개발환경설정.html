개발환경설정

  • 사용하는 개발자 도구 / 툴 : VScode
  • 설치 후 고급시스템 설정 -> 환경변수 -> 위쪽 사용자 환경변수에 path 또는 PATH 에서 편집 클릭
  • path 내용에서 VScode 설치 경로의 bin 폴더까지 추가된 내용이 없다면 추가 후 저장
  • 환경변수 설정 후 명령프롬포트(cmd) 에서 code . 입력하였을 때 VScode가 실행되는지 확인
  • 만약 실행이 안된다면, cmd에 입력하는 명령어를 code나 code. 으로 입력해 볼 것

명령프롬포트(CMD)

  • 명령프롬포트는 UI가 없는 옛날 컴퓨터라고 생각하면 된다
  • cmd에 UI가 없는 이유는 사용자들이 접근할 필요가 없기 때문
  • 우리가 다루게 되는 서버환경도 마찬가지로 사용자가 접근할 필요가 없기 때문에 UI가 없는 것
  • cmd 실행시에 커서가 입력되어있는 위치가 통상은 C:\User\User> 이런식으로 되어있다 이는 현재 내위치
  • 명령프롬포트(cmd)에서 사용되어지는 명령어
    1. dir : 현재 위치에 있는 파일 및 폴더들 보여줌
    2. cd : change directory 위치 옮길 때 사용
    3. mkdir : make directory 새로운 폴더 생성할 때 사용
    4. 입력 도중 탭키(Tab)를 사용하면 자동완성 가능
  • 명령프롬포트로 내가 작업한 폴더까지 이동 후 code . 을 입력해 VScode를 실행하면 VScode에서 폴더열기를 한 것과 같은 효과를 가진다
