Git / Github

  • 깃허브를 사용하는 목적
    1. 내 포트폴리오를 저장하는 공간
    2. 소스코드 공유하는 공간
    3. 프로젝트 협업을 위한 공간
  • 모든 소스코드들을 컴퓨터에 저장하기에는 용량의 한계가 있다
  • 혹은 USB를 분실하거나 컴퓨터에 이상이 생겨 포멧을 하게 된다면 큰일
  • 이런 상황들에 있어 공간이나 장소에 제약 받지 않고 어디서나 사용할 수 있는 것이 바로 깃허브


Git / Github 사용법

  1. 구글에서 github 검색, 회원가입 실시
  2. 회원가입 후 상단메뉴에서 레파지토리 선택하여 new 클릭하여 레파지토리 생성한다
  3. 공개범위는 통상 public으로 한다
  4. 레파지토리가 생성되고 나면 git git git git 으로 이루어진 영어문장들이 나타난다 일단은 대기
  5. 다시 구글로 돌아와서 git을 검색한다
  6. 가장 최상단의 git 사이트로 들어가서 다운로드 선택 후 자신의 운영체제에 맞는 설치파일을 다운받기
  7. 설치에 대하여 특이사항이 없다면 쭉 next 를 클릭해서 설치 해준다
  8. 설치가 끝나고 윈도우버튼, 시작프로그램에 보면 git이라는 폴더가 생겨있고 그 안에 git bash가 있을 것이다
  9. gitbash를 실행하면 명령프롬포트와 비슷한 창이 뜨면서 입력할 수 있는 공간이 생겨있다
  10. git bash에서 입력하는 방법
    1. git config --global user.name "내이름"
    2. git config --global user.email "깃허브 가입시 입력한 이메일 주소"
    3. git config --list 라고 입력 후 마지막에 user.name 과 user.email 이 제대로 입력되어져있다면 성공
  11. 앞서 gitbash에서 설정을 마쳤으면 VScode로 돌아와 내가 만든것들을 커밋해주면 된다
  12. 이 때 폴더열기가 되어 있는 상태여야 한다 폴더가 공유폴더 같이 네트워크상에 공유되는 폴더는 커밋이 불가능
  13. 폴더 여는 방법은 cmd를 통한 방법과 VScode에서 직접 열어주는 방법이 있다
  14. 이 때 폴더열기가 되어있으면 그 안에 있는 내용 전체가 커밋 대상에 포함되어 질 수 잇다
  15. Github로 푸시 하는 방법
    1. VScode상단 메뉴에 터미널 -> 새로운 터미널
    2. VScode 하단에 터미널 창이 띄워지고 사용할 터미널 종류를 Gitbash로 바꿔준다
    3. Gitbash 선택 후 입력창이 나타나는 것을 확인 후, 다음 명령들을 순차적으로 입력하면 된다
    4. -----------------<터미널 입력 방법>-----------------
    5. git init : 초기화 하는 명령어. 최초에 한번 입력하면 된다. 폴더가 바뀌거나 했을 때
    6. git add . : 새로운 파일들 업로드 명령. 원하는 파일 하나만 올리려면 git add 파일이름.확장자
    7. . 이라는 것은 all 을 의미한다
    8. git status : 새롭게 커밋할 수 있는 파일들의 유무를 보여준다
    9. git commit -m "커밋메세지(first commit)" : 커밋에 관한 메세지 입력. 히스토리 남기기
    10. git remote add origin http://~~~~~.git :
      처음 회원가입하고, 깃허브에서 레파지토리 만들면 보이는 창에 주소를 복사해서 붙여넣기 해준다
    11. git remote -v : 연결이 잘 되었는지 확인해주는 명령, 입력한 주소와 같은 주소가 두줄 뜨면 성공
    12. git push origin master : master 브랜치에 푸시 하는 명령
    13. master > master 라는 문구가 보이면 성공
    14. 깃허브 레파지토리에 들어가서 새로고침해보면 정상적으로 레파지토리가 생성되어 있다
  16. 수정사항에 대해서 다시 커밋하는 방법
    1. 코드상에 수정 사항이 생기면 연결된 GitBash가 자동으로 수정사항을 캐치하게 된다
    2. 다시 VScode 상단 메뉴에 터미널 -> 새로운 터미널 Gitbash를 선택
    3. git add . : 추가하는 명령을 입력하면 변동사항이 생긴 파일에 대해서 업로드 준비를 한다
    4. git status : 어떤 파일들에 변동이 생겼는지 보여준다
    5. git commit -m "commit message" : 히스토리에 커밋할 메세지를 입력한다. 한글 안되고 영어로만
    6. git push origin master : master 브랜치에 푸시
    7. master > master 라는 문구가 보이면 성공
    8. 깃허브 레파지토리에 들어가서 새로고침해보면 수정사항이 반영 되어 있다

github로 협업하는 방법

    작업중인 레파지토리의 소스코드 다운 받는 방법
  1. 협업을 하기 위한 레파지토리의 주소가 필요하므로 대상 레파지토리에 가서 초록색 code 버튼을 눌러 주소 복사
  2. 명령프롬포트(cmd)로 돌아와 우리가 다운로드 받을 폴더까지 cd 명령어를 이용해 이동해준다
  3. 이동완료 되었다면, git clone 아까 복사한 주소 -> 의 양식으로 입력
  4. 입력 후 엔터를 치면 리스트가 뜨고 폴더에 이동해보면 레파지토리가 통째로 다운로드 되어있다
    위에서 다운받은 파일들을 작업 한 후 커밋하는 방법
  1. 작업을 마치고 결과물이 나오면 커밋을 해야한다
  2. 앞서 언급한 커밋방법과 거의 동일한데 뒷부분이 조금 다르다
  3. VScode 상단메뉴 터미널 -> 새로운 터미널 -> gitbash -> 입력창이 뜨는 것을 확인 후 진행
  4. git add . : 새로운 추가요소 업로드
  5. git status : 업로드할 파일, 변동요소가 생긴 파일 확인
  6. git commit -m "commit message" : 커밋메세지 입력. 영어로만
  7. git checkout -b 브랜치네임 : 나만을 위한 새로운 브랜치로 들어가는 명령어
    위의 브랜치네임에 새로 생성할 브랜치의 이름을 입력하면 된다
  8. 위 명령을 입력하고 나면 명령줄의 제일 끝 괄호안에 글자가 master에서 내가 입력한 브랜치네임으로 바껴져 있다
  9. git push origin branchName : 새로 생성한 브랜치로 변동사항을 푸시
    커밋 후 깃허브에 와서 작업해야 할 것들
  1. 깃허브 레파지토리에 다시 들어가보면 새로운 브랜치에 글이 올라가있다고 뜬다
  2. 그 다음 compair and pullrequest 버튼을 클릭해 들어가면 입력할 수 있는 페이지가 뜬다
  3. 거기에 메세지를 입력한 후에 초록색 박스 create pullrequest 버튼을 눌러 전송하면 된다
  4. 이 작업 후 마스터브랜치 관리자가 승인을 하면 최종적으로 마스터브랜치에 내 코드가 올라가게 되는 순서이다
    Github를 이용한 협업은 이러한 순환이 이루어진다
  1. 내가 수정한 것을 서브브랜치에 올리고
  2. 그 수정된 부분을 누군가가 받아서 동기화 시키고
  3. 어느 순간 마스터 브랜치에 업로드가 되고
  4. 나도 다시 마스터브랜치를 다운받아 동기화 시키고
  5. 다시 나의 수정본을 서브 브랜치에 업로드
